# Tickit Server Project Guidelines

이 프로젝트는 **Nest.js**, **Prisma**, **PostgreSQL** 기반의 현대적인 백엔드 시스템입니다.
Tickit 데스크탑 앱의 클라우드 확장을 위한 핵심 서버이며, AI 에이전트는 작업을 시작하기 전 본 지침을 준수해야 합니다.

## 📌 규칙 인덱스 (Rule Index)
- **Nest.js 아키텍처**: 모듈화된 구조, 의존성 주입(DI), 계층형 설계 준수.
- **데이터베이스 (Prisma)**: 타입 안정성이 보장된 쿼리 작성 및 효율적인 마이그레이션 관리.
- **코드 컨벤션**: 엄격한 TypeScript 적용, DTO 기반 유효성 검사, 전역 에러 필터 활용.
- **보안 및 인증**: JWT 기반 무상태(Stateless) 인증 및 Passport 전략 활용.
- **Git 워크플로우**: 논리적 단위의 작업 및 명확한 커밋 메시지 (Prefix 사용).

## 🛠 기본 원칙 (Core Principles)
1. **관심사 분리 (SoC)**: Controller는 요청/응답 처리에만 집중하고, 비즈니스 로직은 Service에 작성합니다.
2. **타입 안전성**: `any` 사용을 금지하며, Prisma가 생성한 타입을 적극 활용합니다.
3. **확장성**: 향후 모바일 앱 등 다양한 클라이언트 대응이 가능하도록 RESTful한 API 설계를 지향합니다.
4. **일관성**: 모든 API 응답은 일관된 형식을 유지하며, 적절한 HTTP 상태 코드를 반환합니다.

## 📂 프로젝트 구조 가이드
- `src/auth`: 인증 및 인가 관련 로직
- `src/common`: 전역 필터, 인터셉터, 데코레이터 등 공통 모듈
- `src/reminders`: 리마인더(티켓) CRUD 및 동기화 로직
- `src/users`: 사용자 관리 및 프로필 로직
- `prisma/`: 데이터베이스 스키마 및 마이그레이션 파일
